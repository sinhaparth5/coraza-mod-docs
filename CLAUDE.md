# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run start        # Dev server at http://localhost:3000 (hot reload)
npm run build        # Static build into build/
npm run serve        # Serve the build/ output locally
npm run clear        # Clear Docusaurus cache (.docusaurus/ and build/)
npm run typecheck    # TypeScript type-check (tsconfig.json)
npm run deploy       # Build and push to gh-pages branch
```

Node.js >=20.0 is required.

## Architecture

**Docusaurus v3.10.1** documentation site for **Coraza WAF Mod** (a WAF reverse-proxy project hosted on GitLab). Uses the `classic` preset with `@docusaurus/faster` (Rspack bundler) and `future.v4: true`.

**Deploy mismatch to be aware of:** `editUrl` and `organizationName`/`projectName` point to GitLab (`sinhaparth5/coraza-waf-mod`), but `npm run deploy` pushes to `gh-pages` and `url` in `docusaurus.config.ts` is still the placeholder `your-docusaurus-site.example.com`. Confirm the intended host before running `deploy`.

### Key files

- `docusaurus.config.ts` — site config: title, URL, navbar, footer, theme. Prism syntax highlighting includes `bash`, `yaml`, and `go`.
- `sidebars.ts` — uses `autogenerated` (filesystem ordering via `sidebar_position` frontmatter). Switch to explicit config here when manual ordering is needed.
- `src/css/custom.css` — global CSS and Docusaurus variable overrides. Fonts (Plus Jakarta Sans, JetBrains Mono) loaded via `stylesheets` in config.
- `docs/` — documentation in four sections: `overview/`, `installation/`, `configuration/`, `security/`, plus top-level `faq.md` and `troubleshooting.md`.
- `blog/` — blog posts. Authors in `blog/authors.yml`, tags in `blog/tags.yml`.
- `static/` — files copied verbatim to `build/` root.

### Custom code

- **Swizzled sidebar** — `src/theme/DocSidebar/Desktop/Content/index.tsx` injects `CategoryNav` above the standard sidebar tree. `src/theme/DocSidebar/CategoryNav.tsx` has a **hardcoded `CATEGORIES` array** mapping labels to doc routes (e.g. `Installation → /docs/installation/requirements`). These are not derived from the sidebar config — update them by hand whenever docs are added or renamed.
- **Live roadmap** — `src/pages/roadmap.tsx` renders `src/components/KanbanSection`, which fetches issues client-side from the **GitLab REST API** (`https://gitlab.com/api/v4/projects/sinhaparth5%2Fcoraza-waf-mod`). Open issues are split into "Open" and "In Progress" by the `in-progress` label. Label badge colors are set by regex in `getLabelClass`. The project path is hardcoded in `KanbanSection/index.tsx`.
- **Homepage** — `src/pages/index.tsx` plus `src/components/HomepageFeatures`.

### Content authoring

- Docs support MDX (React components inside Markdown).
- Use `sidebar_position: N` frontmatter to control ordering within a directory.
- `onBrokenLinks: 'throw'` — broken internal links fail the build.
- `tsconfig.json` is IDE/`typecheck`-only; Docusaurus uses Babel/SWC for compilation.
